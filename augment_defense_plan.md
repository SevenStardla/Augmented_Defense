# 증강디펜스 기획서

## 1. 프로젝트 개요

### 게임 제목
**증강디펜스**  
영문 후보: **Augment Defense**

### 장르
2D 타워디펜스 + 로그라이크식 증강 선택

### 한 줄 설명
적의 웨이브를 막으면서 매 라운드마다 랜덤 증강을 선택해 나만의 방어 빌드를 완성하는 전략 디펜스 게임.

### 핵심 콘셉트
일반적인 타워디펜스처럼 타워를 설치하고 적을 막는 구조에,  
웨이브가 끝날 때마다 **증강 카드**를 선택하는 시스템을 추가한다.

플레이어는 매판 다른 증강 조합을 통해 다양한 빌드를 만들 수 있다.

예시:

- 탄환 타워 + 연사 증강 + 관통 증강 = 빠른 다중 타격 빌드
- 냉기 타워 + 빙결 증강 + 빙결 폭발 = 제어형 광역 빌드
- 전기 타워 + 체인 증강 + 감전 피해 = 연쇄 공격 빌드

---

## 2. 핵심 재미

### 2.1 선택의 재미
웨이브 종료 후 3개의 증강 카드 중 1개를 선택한다.  
플레이어는 현재 상황에 맞춰 공격, 제어, 생존, 경제 중 하나를 선택해야 한다.

### 2.2 빌드 조합의 재미
증강은 단독 효과보다 조합되었을 때 더 강력해진다.

예시:

```text
냉기 타워 둔화 증가
+ 빙결 확률 증가
+ 빙결된 적 사망 시 폭발
= 적을 묶어두고 터뜨리는 빙결 폭발 빌드
```

### 2.3 반복 플레이의 재미
랜덤 증강 시스템으로 매판 다른 플레이 경험을 만든다.

```text
↓

↓

↓

↓

↓
클리어 또는 게임 오버
```

## 4. 세계관 콘셉트

### 배경
붕괴된 도시 한가운데, 인류의 마지막 방어 코어가 남아 있다.  
플레이어는 방어 시스템 관리자이며, 몰려오는 적을 막기 위해 방어 타워를 설치하고 전투 중 얻은 증강 데이터를 활용해 방어 체계를 진화시킨다.

### 플레이어 역할
- 방어 시스템 관리자
- 타워 설치자
- 증강 선택자
- 마지막 코어의 수호자

### 적 콘셉트
- 오염된 데이터 생명체
- 기계 괴물
- 돌연변이 개체
- 방어막을 가진 강화형 적
- 보스급 침입체

## 5. 주요 시스템

## 5.1 코어 시스템

코어는 플레이어가 지켜야 하는 핵심 오브젝트다.

### 기본 규칙
- 적이 코어에 도착하면 코어 체력이 감소한다.
- 코어 체력이 0이 되면 게임 오버.
- 코어는 증강을 통해 강화할 수 있다.

### 코어 기본 능력치 예시
| 항목 | 값 |
|---|---:|
| 최대 체력 | 100 |
| 적 1마리 도착 시 피해 | 10 |
| 보스 도착 시 피해 | 30 |

### 코어 증강 예시
| 증강 이름 | 효과 |
|---|---|
| 강화 장갑 | 코어 최대 체력 +30 |
| 긴급 수리 | 웨이브 종료 시 코어 체력 5 회복 |
| 방어막 생성기 | 웨이브 시작 시 보호막 15 획득 |
| 최후의 저항 | 코어 체력 30% 이하일 때 모든 타워 공격력 +20% |
| 1회 부활 | 코어가 파괴될 때 1회 체력 30으로 부활 |

## 5.2 타워 시스템

타워는 적을 자동으로 공격하는 방어 수단이다.

### MVP 타워 종류
초기 구현에서는 타워 3종을 목표로 한다.

---

### 1. 탄환 타워

가장 기본적인 단일 공격 타워.

| 항목 | 설명 |
|---|---|
| 역할 | 안정적인 단일 딜 |
| 장점 | 범용성 높음 |
| 단점 | 물량 처리 약함 |
| 추천 증강 | 연사, 치명타, 관통 |

#### 기본 능력치 예시
| 능력치 | 값 |
|---|---:|
| 공격력 | 10 |
| 공격속도 | 1초 |
| 사거리 | 4 |
| 비용 | 50 |

#### 증강 예시
- 공격력 +20%
- 공격속도 +25%
- 치명타 확률 +10%
- 탄환 관통 +1
- 체력이 높은 적 우선 공격 가능

---

### 2. 냉기 타워

적을 느리게 만드는 제어형 타워.

| 항목 | 설명 |
|---|---|
| 역할 | 둔화, 빙결 |
| 장점 | 적 이동 제어 |
| 단점 | 기본 피해량 낮음 |
| 추천 증강 | 둔화 강화, 빙결, 빙결 폭발 |

#### 기본 능력치 예시
| 능력치 | 값 |
|---|---:|
| 공격력 | 5 |
| 공격속도 | 1.2초 |
| 사거리 | 3.5 |
| 둔화율 | 20% |
| 비용 | 60 |

#### 증강 예시
- 둔화율 +15%
- 둔화 지속시간 증가
- 일정 확률로 적 빙결
- 빙결된 적 추가 피해
- 빙결된 적 사망 시 폭발

---

### 3. 전기 타워

여러 적에게 공격이 튕기는 연쇄 공격 타워.

| 항목 | 설명 |
|---|---|
| 역할 | 다중 타격, 물량 처리 |
| 장점 | 적이 많을수록 강함 |
| 단점 | 단일 보스 딜 약함 |
| 추천 증강 | 체인 증가, 감전, 감전 확산 |

#### 기본 능력치 예시
| 능력치 | 값 |
|---|---:|
| 공격력 | 7 |
| 공격속도 | 1.5초 |
| 사거리 | 4 |
| 체인 횟수 | 2 |
| 비용 | 70 |

#### 증강 예시
- 체인 횟수 +1
- 감전 지속 피해 추가
- 감전된 적이 받는 피해 증가
- 감전 적 처치 시 주변 번개
- 체인 피해 감소율 완화

적은 정해진 경로를 따라 이동하며 코어를 향해 간다.

### MVP 적 종류

특징:
- 빠르게 코어로 접근

특징:

| 골드 보상 | 50 |

- 10웨이브에서 등장

- 일정 시간마다 이동속도 증가
- 주변 적에게 보호막 부여

- 코어 도착 시 큰 피해

## 5.4 웨이브 시스템

### 기본 규칙
- 웨이브 시작 버튼을 누르면 적이 순서대로 등장한다.
- 모든 적을 처치하거나 코어에 도착하면 웨이브 종료.
- 웨이브 종료 후 증강 카드 선택 화면이 열린다.
- 총 10웨이브를 목표로 한다.
- 10웨이브에는 보스가 등장한다.

### 웨이브 예시

| 웨이브 | 등장 적 | 특징 |
|---:|---|---|
| 1 | 일반 적 8마리 | 기본 조작 학습 |
| 2 | 일반 적 12마리 | 타워 배치 테스트 |
| 3 | 일반 적 10마리, 빠른 적 4마리 | 빠른 적 등장 |
| 4 | 일반 적 15마리, 탱커 2마리 | 탱커 등장 |
| 5 | 빠른 적 10마리, 일반 적 10마리 | 속도 대응 |
| 6 | 방어막 적 5마리, 일반 적 15마리 | 보호막 적 등장 |
| 7 | 탱커 5마리, 빠른 적 8마리 | 혼합 웨이브 |
| 8 | 방어막 적 8마리, 빠른 적 12마리 | 중후반 압박 |
| 9 | 일반 적 20마리, 탱커 6마리, 방어막 적 5마리 | 최종 전 웨이브 |
| 10 | 보스 1마리, 일반 적 다수 | 보스 웨이브 |

## 5.5 증강 시스템

증강은 이 게임의 핵심 시스템이다.

### 기본 규칙
- 웨이브 종료 후 증강 카드 3장이 등장한다.
- 플레이어는 3장 중 1장을 선택한다.
- 선택한 증강은 즉시 적용된다.
- 같은 증강이 중첩될 수 있는지 여부는 증강별로 정한다.

### 증강 분류

#### 1. 타워 증강
타워의 공격력, 공격속도, 사거리, 특수 능력을 강화한다.

예시:
- 탄환 타워 공격력 +20%
- 냉기 타워 둔화율 +15%
- 전기 타워 체인 횟수 +1
- 모든 타워 공격속도 +10%

#### 2. 속성 증강
공격에 화상, 독, 감전, 빙결 같은 상태 이상을 부여한다.

예시:
- 공격 시 10% 확률로 화상
- 감전된 적이 받는 피해 +15%
- 빙결된 적 사망 시 폭발
- 독 피해 중첩 가능

#### 3. 코어 증강
코어의 체력, 보호막, 회복, 위기 대응 능력을 강화한다.

예시:
- 코어 최대 체력 +30
- 웨이브 종료 시 코어 체력 회복
- 코어 주변 적에게 지속 피해
- 코어가 한 번 죽을 피해를 막고 부활

#### 4. 경제 증강
골드 획득량, 설치 비용, 판매 환급률 등을 강화한다.

예시:
- 적 처치 골드 +20%
- 웨이브 클리어 보상 증가
- 타워 설치 비용 감소
- 타워 판매 환급률 증가

## 7. 맵 기획

### MVP 맵
첫 버전은 맵 1개만 구현한다.

### 기본 구조
- 적 시작 지점
- 적 이동 경로
- 코어 위치
- 타워 설치 가능 타일
- 설치 불가 타일

### 맵 형태
처음에는 너무 복잡하지 않은 S자 경로를 추천한다.

```text
[적 시작]
   ↓
→ → → ↓
↑     ↓
↑ ← ← ↓
      ↓
    [코어]
```

### 확장 맵 후보
| 맵 이름 | 특징 |
|---|---|
| 폐허 도시 | 기본 맵, 단순한 경로 |
| 냉각 지대 | 일부 구역에서 적 이동속도 감소 |
| 오염 공장 | 특정 구역에서 적이 지속 피해를 받음 |
| 전력 시설 | 전기 타워 강화 구역 존재 |

## 14. 포트폴리오에서 강조할 점

### 1. 웨이브 시스템
적 스폰, 웨이브 진행, 클리어 판정, 난이도 증가 구조를 설명할 수 있다.

### 2. 타워 탐색 시스템
가장 가까운 적, 가장 앞선 적, 체력이 높은 적 등 타겟팅 방식을 구현하면 좋다.

### 3. 증강 시스템
웨이브 종료마다 게임 규칙이 바뀌는 구조를 설명할 수 있다.

### 4. 데이터 기반 설계
타워, 적, 증강, 웨이브 데이터를 ScriptableObject로 관리하면 포트폴리오 가치가 올라간다.

### 5. 오브젝트 풀링
총알, 적, 데미지 팝업에 오브젝트 풀링을 적용하면 최적화 경험으로 설명하기 좋다.

## 15. 구현 시 주의할 점

### 15.1 기능을 한 스크립트에 몰아넣지 않기
GameManager가 모든 기능을 직접 처리하지 않도록 한다.

나쁜 예:
```text
GameManager가 적 생성, 타워 공격, UI, 골드, 증강까지 전부 처리
```

좋은 예:
```text
GameManager: 게임 상태 관리
WaveManager: 웨이브 관리
EconomyManager: 골드 관리
TowerPlacement: 타워 설치 관리
AugmentManager: 증강 관리
```

### 15.2 MVP를 먼저 완성하기
처음부터 플레이어 스킬, 여러 맵, 다양한 보스 패턴을 넣지 않는다.

먼저 완성해야 할 것:
- 적 이동
- 타워 공격
- 코어 체력
- 웨이브 진행
- 증강 선택

### 15.3 증강은 처음부터 너무 복잡하게 만들지 않기
초기 증강은 단순한 수치 증가 중심으로 만든다.

초기 추천:
- 공격력 증가
- 공격속도 증가
- 사거리 증가
- 골드 증가
- 코어 체력 증가

이후 추가:
- 빙결 폭발
- 체인 라이트닝
- 독 중첩
- 감전 확산

## 16. 개발 체크리스트

### 기본 전투
- [ ] 적이 경로를 따라 이동한다.
- [ ] 적이 코어에 도착하면 코어 체력이 감소한다.
- [ ] 적이 죽으면 제거된다.
- [ ] 적을 처치하면 골드를 얻는다.
- [ ] 코어 체력이 0이 되면 게임 오버가 된다.

### 타워
- [ ] 타워를 설치할 수 있다.
- [ ] 설치 시 골드가 차감된다.
- [ ] 설치 불가능한 위치에는 설치되지 않는다.
- [ ] 타워가 사거리 내 적을 탐색한다.
- [ ] 타워가 적을 자동으로 공격한다.
- [ ] 타워를 판매할 수 있다.

### 웨이브
- [ ] 웨이브 시작 버튼이 있다.
- [ ] 웨이브마다 적이 순서대로 등장한다.
- [ ] 모든 적이 처리되면 웨이브가 종료된다.
- [ ] 웨이브가 끝나면 증강 선택으로 넘어간다.
- [ ] 10웨이브에 보스가 등장한다.

### 증강
- [ ] 웨이브 종료 후 증강 카드 3장이 나온다.
- [ ] 카드 1장을 선택할 수 있다.
- [ ] 선택한 증강 효과가 적용된다.
- [ ] 선택 후 다음 웨이브로 넘어간다.
- [ ] 중복 가능 여부를 처리한다.

### UI
- [ ] 코어 체력이 표시된다.
- [ ] 골드가 표시된다.
- [ ] 현재 웨이브가 표시된다.
- [ ] 선택한 타워 정보가 표시된다.
- [ ] 증강 카드 UI가 표시된다.
- [ ] 게임 오버 UI가 표시된다.
